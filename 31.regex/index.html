Forget everything you've heard about regex looking scary. We're going to build up name, email, phone, and CNIC validation one tiny piece at a time — nothing more than needed.

1 What a regex actually does

A regex is just a short description of a "shape" of text. That's it. Nothing scarier than that.

Plain search vs. shape search

Plain search: "find the exact word cat"

Regex (shape) search: "find 4 digits in a row" — it doesn't care which 4 digits, only that the shape matches.

How it looks in JavaScript
/cat/a regex is written between two slashes

Whatever is between the slashes is the pattern. We'll build patterns up slowly from here.

2 Idea 1 — matching a digit: \d

Phone numbers and CNICs are mostly digits. So the very first thing we need is a way to say "a digit goes here."

The rule

\d means one single digit (0 through 9). Nothing else.

/\d/looking inside → Room 42, door B

It found the first digit and stopped, because \d by itself only asks for one digit.

Also useful
Symbol Means one of
\d a digit (0-9)
\w a letter, digit, or underscore
\s a space, tab, or line break

For name/email/phone/CNIC validation, you'll mostly reach for \d. Keep the other two in the back of your mind.

3 Idea 2 — repeating it: +, {n}

A phone number isn't 1 digit, it's several. We need a way to say "repeat this."

One or more
/\d+/→ order #8421

+ after something means "one or more of it, as many as there are."

An exact count
/\d{7}/→ exactly 1234567

{7} means "exactly 7 of the thing before it." You can also write {2,4} to mean "between 2 and 4 times."

Quick reference
Symbol Means
+ one or more
* zero or more
? zero or one (optional)
{n} exactly n times
{n,m} between n and m times

4 Idea 3 — checking the whole thing: ^ and $

Big problem: /\d{5}/ matches 5 digits hiding anywhere in a string — even inside a much longer, invalid one. For form validation we need the entire input to match, not just a piece of it.

The problem
/\d{5}/→ abc12345xyz999

This "matches" even though the input is clearly not just 5 digits. Not good enough for validating a form field.

The fix

^ means "start of the text". $ means "end of the text".

/^\d{5}$/→ "12345" ✓ matches
/^\d{5}$/→ "abc12345xyz" ✗ no match

Read ^\d{5}$ as: "from the very start to the very end, this must be exactly 5 digits — nothing before, nothing after." This is the single most important trick for form validation.

5 Idea 4 — matching letters: [a-zA-Z]

A name field needs letters, not digits. Square brackets [ ] let you say "match one character from this set."

Letter ranges
Pattern Matches one
[a-z] lowercase letter
[A-Z] uppercase letter
[a-zA-Z] letter, either case
[a-zA-Z\s] letter, either case, OR a space (for full names)
Building a name pattern
start with [a-zA-Z\s] — one letter or space
[a-zA-Z\s]+ — one or more of them
^[a-zA-Z\s]+$ — the whole field must be only letters/spaces

Final pattern: ^[a-zA-Z\s]+$ — matches "Hassan Ali", rejects "Hassan123".

6 Putting it together — 3 real patterns

You now know everything needed: \d, +/{n}, ^ $, and [a-zA-Z]. Let's assemble the real ones, piece by piece.

Phone — 03XX-XXXXXXX
the number always starts with 03
03\d{2} — 2 more digits (the XX)
03\d{2}- — a literal dash
03\d{2}-\d{7} — 7 more digits
^03\d{2}-\d{7}$ — whole field only

Final: ^03\d{2}-\d{7}$ — matches "0300-1234567".

CNIC — XXXXX-XXXXXXX-X
\d{5} — 5 digits
\d{5}-\d{7} — dash, then 7 digits
\d{5}-\d{7}-\d{1} — dash, then 1 digit
^\d{5}-\d{7}-\d{1}$ — whole field only

Final: ^\d{5}-\d{7}-\d{1}$ — matches "35202-1234567-1".

Email — the beginner-friendly version
\w+ — one or more letters/digits (the name part)
\w+@ — a literal @ symbol
\w+@\w+ — the domain name, e.g. "gmail"
\w+@\w+\.[a-zA-Z]{2,} — a dot, then 2+ letters (e.g. "com")
^\w+@\w+\.[a-zA-Z]{2,}$ — whole field only

Final: ^\w+@\w+\.[a-zA-Z]{2,}$ — matches "ali@gmail.com". (Note: . normally means "any character," so a real dot is written as \.)

7 Using a pattern in code

A pattern alone does nothing — you need to ask JavaScript to check it. Two methods cover almost every case.

.test() — yes or no

Use this for form validation: "is this input valid?"

/^\d{5}$/.test("12345")→ true
.match() — what actually matched

Use this when you want to pull the matching text out, not just true/false.

"call 0300-1234567".match(/\d+/)→ ["0300"]
That's genuinely the whole toolkit: \d, \w, \s · [a-zA-Z] · + and {n} · ^ and $ · .test() and .match(). Everything else in regex is an extra convenience built on these same ideas.
